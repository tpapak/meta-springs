Standard NMA convention: yij = μi + dj + δij + ε, so positive δij pulls the tau node toward larger outcome values. Each study has one y-row per arm; Si is a vertical orange bar at the centroid of its arm-treatment columns spanning those rows, so each τ-spring is horizontal at the same y as the corresponding part of Si. Black dots = tau (BLUP) nodes at x = xSi + scale·δij; τ-chain coloured by chain length N (blue→cyan→amber→red). Obs springs are horizontal, tau node → treatment column. Right panel = posterior on τ² (flat prior); bars ∝ exp(ln Zk − max).
Why some BLUPs may cross the centroid: δij = (τ²/(vij+τ²))·(yij − μ̂i − dj), so the side a tau node ends up on is dictated by the residual of the data against the network MAP, not by which side the treatment column sits on. With τ²>0, having every BLUP land on its "natural" side requires every within-study contrast to agree with the network estimate — which contradicts heterogeneity. Crossings are therefore an inherent feature of inconsistent data: the model is honestly absorbing the inconsistency into δij.
